My first hexo blog about how to deploy hexo

发表于2019-12-05|更新于2024-02-04|hexo
|阅读量:

为什么写博客、

在学生生活中，无聊是最为恐怖的事情，那么怎么避免太过无聊的事情呢，当然是写点什么记录自己的生活，但是日记又太过无聊（我的字丑），而跟风写一下博客就还行。

怎么配置hexo博客

写博客当然是要找一个教程比较多，起点低的方法，其中hexo博客要求很低（windows下面，有一个github账号就可以完成相应的配置），其次hexo的工作流也是一个和未来找工作比较相关工作流（git流程），所以使用hexo进行相应的配置。下面是正式配置过程

环境安装

我是在windows下进行的配置，所以相对来说都是有界面的比较简单，总的来说需要以下几个windows不自带的东西

  1. git
  2. npm
  3. ssh（使用的putty）
  4. 相应的markdown编辑软件（我用的vscode+markdown all in one）
    下面进行相应的安装介绍

git 安装

简单来说就是按照官方给的教程一步步来安装git,因为git之前我已经搞过了，所以忘了具体的流程，需要注意到的是windows下的不一定会自动添加路径，有可能需要自己添加，如何检查自己的git是否安装好，只需要打开powershell（powershell是不需要折腾的windows最好的命令行软件），输入：git要是提示不是可执行的命令行，那么就应该去找一下路径手动配置一下，总的来说是简单的。

npm安装

这或许就是windows广泛使用的原因了，就是基本提供了可视化的才是windows的软件，npm就是下载安装就完事，然后里面提供了加入到path的方法，所以点击了就可以完成。同样的在powershell里面输入：npm 要是没问题就完事。

putty ssh客户端的安装

简单来说就是下载了解压然后加入路径完事。

vscode markdown 插件安装

这个就超级简单了，插件主页，这就是vscode厉害的地方了，找了一帮免费大佬写插件，懂吧，我感觉vscode会一统天下。

正式配置hexo

这里面主要借鉴了两个人的hexo配置教程

  • 教程1 不说了，暂时还没搞完这里面的所有东西，只是完成了配置的这一步
  • 教程2 这个比较基础，我推荐这两个结合起来看。

总的来说就是这个两个教程配置部分主要是两个步骤：

  1. 完成本地的hexo博客配置
  2. 完成github上的博客设置
  3. 完成ssh和github上的配置

基本上就是这三个部分，其中1、2可以替换，但是ssh和github部分的就是有点问题，由于使用的是putty的ssh-keygen进行的密钥生成，会出现一些问题，这里先不谈了。
