Spring装配bean的方法
spring
发布日期:   2020-01-10

Spring bean

Spring 核心容器主要以Beans 作为功能实现，并且通过Beans 实现控制反转，使用配置文件xml文件通过反射机制使用BeanFactory实现Bean自动装配。除此之外还提供了Bean 实例缓存、生命周期管理、 Bean 实例代理、事件发布、资源装载等高级服务。

实现流程

Spring 启动时读取应用程序提供的 Bean 配置信息，并在 Spring 容器中生成一份相应的 Bean 配置注册表，然后根据这张注册表实例化 Bean，装配好 Bean 之间的依赖关系，为上层应用提供准备就绪的运行环境。 其中 Bean 缓存池为 HashMap 实现。

这里面配置Bean的主要方法有以下三类：

  • Xml配置

  • Configuration 注解的类

  • AutoWired 自动注入注解

Xml配置方法

Xml配置使用Xml定义

bean id="User" class="com.lixiande.enities.User" p:name="zhangyukang" autowire="byType" scope="prototype" ></bean>

这里面主要涉及Bean的id、class、类的内部参数定义、生成的Bean的作用域、以及自动注入的方式。这里面就参数进行介绍：

  1. autowire：注入的方式，主要有byName和byType方式以及constructor方法，分别对应按照bean的id配置、类型配置和具有相同的属性的Bean的。

  2. id： 对应的bean的id。id作为这个bean的名字，是唯一的不可重复的，在使用autowire="byName"形式时候会自动查找具有相同id的bean自动装填。

  3. class： class定义了bean加载的类型，使用xml方式配置时候使用XmlBeanFactory按照class的路径调用getBean(String beanName)方法获得对应的Bean。从某种意义上说就是反射的class.forName()形式。

  4. scope: 定义xml生成的Bean的作用域，分别由singleton、prototype等形式。

    除此之外还有使用很多详细的参数，如p:name的形式对应bean内部的构造函数使用的参数设置，以及内部的很多参数设置的标记如ref property entry等。

    Configuration 注解

    Configuration 注解使用在class前面，定义该类为一个配置类，定义Bean间的依赖关系和Bean的构造形式，下面以代码示例

    @Configuration
public class EntityConfig{
    //定义单个Bean的形式
    @Bean
    public User user(){
        return new User();
    }
    //多个类的依赖关系，这里的User参数会通过自动注入实现
    @Bean
    public Company company(User user){
        return new Company();
    }
}

    使用@Bean 注解获得对应类型的Bean，调用对应得构造函数，这个方法相对来说比较容易显示定义配置关系，但是相对于xml形式代码量会比较大。除此之外这个配置形式数据也可以看作是一个Bean，可以通过Xml进行配置。

    AutoWired 自动注入注解

    AutoWired 自动注入注解主要使用@AutoWired 注解实现配置也就是通过定义了的Bean，使用@AutoWired 而不直接显示定义获取形式可以将Bean写在类变量中。

    public class AppConfig{
 //这里会自动生成一个User类型的数据
 @AutoWired
 private User user;
 @Bean
 public Company company(){
     return new Company(user);
 }
}
