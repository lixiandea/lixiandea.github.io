java Synchronized 锁

Synchronized关键字是Java关键字，用于保证方法或者代码块在执行时，同一时刻多个线程只能有一个线程执行该方法，同时也可以保证多线程中共享变量的内存可见性，Java中每一个对象都可以作为锁，这就是Synchronized实现同步的基础。

使用方法

  • 修饰代码块，被修饰的代码块称为同步代码块，起作用范围是{}中的代码，作用对象是调用这个代码块的所有对象。
  • 修饰一个方法，和代码块的形式类似但是该方法称为同步方法，作用对象是调用这个方法的对象。
  • 修饰静态方法，其作用范围是整个静态方法，作用对象是类的所有对象。

使用示例

修饰代码块
class Solution implements Runnable{
    public static int count = 0;
    @Override
    public void run() {
        synchronized (this){
            for(int i = 0; i<75; i++)
            {
                try {
                    System.out.println(Thread.currentThread().getName() + ":" +(count++) );
                    Thread.sleep(100);
                }
                catch (Exception e)
                {
                    e.printStackTrace();
                }
            }
        }
    }

    public static void main(String[] args){
        Solution s= new Solution();
        Thread t1 = new Thread(s,"Thread1");
        Thread t2 = new Thread(s,"Thread2");
        t1.start();
        t2.start();
    }
}

public synchronized void method(){

同步方法

和修饰内部代码块相同。但是需要额外以下几点：

  • 父类定义了同步方法，子类需要额外加上synchronized关键字才能将子类对应方法定义为同步方法。
  • 接口方法不能使用synchronized修饰，也就是不能对接口强制要求同步。
  • synchronized 方法存在一定的性能缺陷。
  • 若将run()方法声明为synchronized，会导致本类中的其他任何同步方法都不能成功执行，所以在run()方法中只能使用同步代码块。

同步静态方法

除此之外还可以使用synchronized修饰静态方法，也就是

public synchronized start void method(){
    //todo
}

由于静态方法属于类不属于对象，所以同步静态方法锁定这个类的所有对象，所以所有的对象都会共享同一把锁。

class Solution implements Runnable{
    private volatile static int count = 0;
    @Override
    public void run() {
        method();
        //System.out.println(Thread.currentThread().getName()+"非同步:" + count);
    }
    public synchronized static void method() {

            for (int i = 0; i < 5; i++) {
                try {
                    System.out.println(Thread.currentThread().getName() + " count :" + (count++));
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }

        }
    }

    public static void main(String[] args){

        Solution s1= new Solution();
        Solution s2= new Solution();
        Thread t1 = new Thread(s1,"Thread1");
        Thread t2 = new Thread(s2,"Thread2");
        t1.start();
        t2.start();
    }
}

hashmap

hashmap 是用于做键值对处理的数据类型，使用红黑树作数据结构和扩容优化，主要的继承关系为在java.util.Map下面还有HashMap、LinkedHashMap、TreeMap、HashTable等数据结构。

HashMap数据形式

根据HashCode值存储数据，大多数（不发生hash冲突的情况）下能够直接定位到它的值，具有很快的访问速度。具有以下特征：

  • 遍历顺序不定，遍历得到的序列不是固定的。

  • 最多只允许一条记录的键为空。

  • 非线程安全，任一时刻多个县城同时写hashmap可能带来数据不一致，若要线程安全，需要使用Collection中的synchronizedMap方法使HashMap具有线程安全能力，或者使用ConcurrentHashMap.

    存储结构

    结构实现上来说，HashMap使用数组+链表+红黑树实现。其中主要的数据结构hash桶数组,每一个桶代码如下：

    static class Node implements Map.Entry {
      final int hash;    //用来定位数组索引位置
      final K key;
      V value;
      Node next;   //链表的下一个node

      Node(int hash, K key, V value, Node next) { ... }
      public final K getKey(){ ... }
      public final V getValue() { ... }
      public final String toString() { ... }
      public final int hashCode() { ... }
      public final V setValue(V newValue) { ... }
      public final boolean equals(Object o) { ... }
}

    Node实现了Entry()接口，本质上是一个映射关系。HashMap使用Hash表进行存储，HashMap使用链地址法在每一个数组元素上加链表结构，但数据Hash后将数据放在对应的下标元素链表上。如下：

    每一个键对应一个链表，每当对应的键值对插入HashMap，将值插入到对应键的hashCode()的HashCode，再进行高位运算和取模运算定位键值对的储存位置，当HashCode相同的时候即发生Hash冲突，Hash冲突概率越小，存取效率越高。也就是桶越多读取效率越高，但是空间开销也会越大。实际情况下需要好的Hash算法和扩容机制。

Java中的HashMap主要有以下字段：

int threshold;             // 所能容纳的key-value对极限 
final float loadFactor;    // 负载因子
int modCount;  
int size;

在初始化时HashMap中的桶数组长度为16，LoadFactor是0.75，而Threshold则为最大能容纳的键值对。在定义好长度之后，负载因子越大则能容纳的键值对越多。当插入数据使HashMap长度超过阈值之后就进行resize，将容量拓展为原来的两倍，在实际应用中可以考虑修改LoadFactor得到更好的储存性能或者更好的查找性能。但是当数据长度依旧超长的时候，引入红黑树进行优化。当一个桶的元素超过TREEIFY_THRESHOLD，就执行treeifyBin()用红黑树替换链表，从而提高速度。整体逻辑如下：
